## Editorial Rule

The wiki should summarize and route. Detailed procedures belong in the repository guide so they can be reviewed with source, UI evidence, and pull-request checks.

## How To Read This Wiki

Use the wiki to find the right product area, then follow the repository guide for detailed steps. If a wiki page links to Known Limitations, treat that limitation as part of the workflow. A feature should not be considered customer-ready merely because it appears in navigation or has a draft wiki page.
