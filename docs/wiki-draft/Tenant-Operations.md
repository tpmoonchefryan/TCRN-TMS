# Tenant Operations

Regular tenant operations cover tenant workspace navigation, organization structure, tenant settings, tenant users/roles, integrations, and talent scopes.

## Operating Principle

Always confirm the active workspace, tenant, subsidiary, and talent scope before interpreting data or changing settings.

## Best-Practice Summary

Use tenant operations for business configuration inside one tenant. Keep platform health checks in AC and talent campaign work in the talent workspace.

Example: when preparing a regional subsidiary, confirm tenant and subsidiary scope first, then review organization/business settings and user access. Escalate to AC only if tenant provisioning or cross-tenant configuration is wrong.

## Current Caveat

Tenant security and integration routes still have open proof limitations. Do not promote create/import/delete/batch/test procedures until focused proof exists.

Detailed guide: [Tenant Admin](../user-guide/tenant-admin/README.md).
