# Talent Workspace Overview

Talent workspaces are scoped to creator operations such as customer views, reports, public presence, and settings.

## Current Boundary

Customer, report, and settings routes are accepted only for the represented G13 proof slices. Use the active Known Limitations page for any sensitive mutation, PII, or cross-scope claim that was not part of those slices.

## Best Practice Summary

Use talent workspace for creator-scoped customer, report, settings, and campaign context. Escalate to tenant admin only when the task changes tenant-wide configuration or access.

Example: review customer/report context in the talent workspace before editing a public campaign page; do not request tenant-wide admin rights for a talent-scoped review.

Detailed guide: [Talent Workspace](../user-guide/talent-workspace/README.md).
